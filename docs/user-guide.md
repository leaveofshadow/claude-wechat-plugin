# WeChat Remote Control — User Guide

## 快速开始

### 1. 首次登录

```
/wx-login
```
扫描二维码，微信确认登录。

### 2. 连接微信到当前窗口

```
/wx-on
```
注册轮询 Cron，当前窗口开始接收微信消息。

### 3. 用微信发送指令

在手机微信中给 Bot 发消息，例如：
- "查看当前项目状态"
- "运行测试"
- "切换 thesis"

## Skills 命令参考

### `/wx-login`
登录或重新登录微信 Bot。

- 自动检测是否已登录
- 显示二维码供微信扫码
- 登录后发送测试消息验证

### `/wx-on`
将微信连接到当前 Claude Code 窗口。

- 检查残留 Cron，如有则清理
- 扫描当前项目
- 注册每分钟轮询 Cron（durable，重启后保留）
- 默认免审模式
- 发送微信确认消息

**注意**：同一时刻只有一个窗口能轮询微信。切换窗口前先在旧窗口 `/wx-off`。

### `/wx-off`
断开微信连接。

- 删除所有微信轮询 Cron
- 发送微信通知告知已断开
- 新窗口可运行 `/wx-on` 接管

### `/wx-status`
查看微信系统全面状态。

检查项：
| 检查项 | 说明 |
|--------|------|
| Bot 登录 | token 是否有效 |
| API 连通 | poll 是否正常 |
| 当前项目 | 活跃项目名 + 工作目录 |
| Plan 审批 Hook | timeout 值 |
| 工具审批 Hook | timeout 值 |
| 提问审批 Hook | timeout 值 |
| 守护进程 | 不应有残留 |
| Cron 任务 | 是否在运行 |

### `/wx-switch [项目key]`
切换微信交互的活跃项目。

- 无参数：显示当前活跃项目和可用列表
- `<key>`：切换到指定项目
- 切换后默认免审
- 发送微信确认

项目切换也可在微信中发送 `切换 <项目名>`。

### `/wx-projects`
查看和扫描可用项目。

- 自动从 `~/.claude/projects/` 扫描最近 30 天有活动的项目
- 显示项目列表，`*` 标记当前活跃项目

### `/wx-approve [on/off/部分]`
管理工具审批模式。

| 命令 | 效果 |
|------|------|
| `/wx-approve 开启` | 全部工具需审批（Bash/Edit/Write/Agent） |
| `/wx-approve 关闭` | 免审模式（默认） |
| `/wx-approve 部分` | 选择性审批（如只审批 Edit 和 Write） |
| `/wx-approve 查看` | 查看当前配置 |

**注意**：修改立即生效，不需要重启。Plan 审批和提问审批不受此配置影响。

### `/wx-cron [注册/续期/查看/删除]`
管理微信轮询 Cron。

- **注册**：创建新的每分钟轮询 Cron
- **续期**：删除旧 Cron 并创建新的（7 天自动过期前续期）
- **查看**：显示当前 Cron 状态
- **删除**：停止 Cron

## 微信端指令

在微信中发送消息，Cron 轮询到后自动执行：

| 指令 | 说明 |
|------|------|
| 任意文本 | 作为 Claude Code 指令执行 |
| `切换 <项目key>` | 切换活跃项目 |
| `y` / `是` / `ok` | 审批通过（在审批等待中） |
| `n` / `否` / `取消` | 审批拒绝（在审批等待中） |
| `A` / `B` / `C` / `D` | 选择提问选项 |

---

## 通知类型

| 通知 | 触发条件 | 含义 |
|------|---------|------|
| `[Claude 已完成]` | Claude 结束回复 | 任务完成，可发新指令 |
| `[Claude 需要审批]` | 工具执行被拦截 | 需回复 y/n（2分钟无回复自动批准） |
| `[Claude 方案审批]` | Plan 提交 | 需回复 y/n（3分钟无回复自动批准） |
| `[Claude 提问]` | Claude 提问 | 回复选项字母或数字 |
| `[已批准]` | 审批通过确认 | 工具/方案开始执行 |
| `[已拒绝]` | 审批拒绝确认 | 操作已取消 |
| `[已收到]` | 收到微信回复 | 选择已自动输入 |

## 工作流示例

### 日常远程开发

```
1. 终端: /wx-on
   → 微信收到 "[SkillHub] 微信已连接"

2. 微信发送: "看看 server 有什么编译错误"
   → Claude 检查并返回结果

3. 微信发送: "修复那个编译错误"
   → Claude 修复代码，完成后推送通知

4. 微信发送: "运行测试"
   → Claude 运行测试，推送结果
```

### 切换项目

```
微信发送: "切换 thesis"
→ 收到: "已切换到项目: 论文助手"

微信发送: "帮我整理文献综述的结构"
→ 在论文助手项目上下文中执行
```

### 开启审批模式

```
终端: /wx-approve 开启

微信收到: "[Claude 需要审批] Edit: src/main.py"
微信回复: "y"
→ 收到: "[已批准] 工具执行中..."
```

### Cron 续期

```
终端: /wx-cron 续期
→ 删除旧 Cron，创建新 Cron
→ 新 Cron 7 天有效
```

## 故障排查

### 微信收不到消息

1. 运行 `/wx-status` 检查状态
2. 检查 Bot 是否登录：`cat ~/.mcp-wechat-server/account.json`
3. 在手机上给 Bot 发条消息测试
4. 如果仍不通，运行 `/wx-login` 重新登录

### 消息发不出去 / 不被执行

1. 确认 Cron 在运行：`/wx-cron 查看`
2. 检查是否有多窗口竞争：运行 `/wx-off` 后重新 `/wx-on`
3. 检查活跃项目是否正确：`/wx-switch`

### 审批一直等待

1. 检查审批配置：`/wx-approve 查看`
2. 确认当前窗口是活跃项目：`/wx-status`
3. 如果不需要审批：`/wx-approve 关闭`

### Cron 过期

Cron 7 天自动过期。过期后：
1. 运行 `/wx-cron 续期` 重新注册
2. 或重新运行 `/wx-on`

## 注意事项

- **一个窗口原则**：同一时刻只有一个窗口应运行微信 Cron，否则消息会被随机消费
- **Cron 仅空闲触发**：轮询只在 REPL 空闲时执行，不会中断正在执行的操作
- **审批超时**：工具审批 2 分钟、Plan 审批 3 分钟无回复自动批准
- **免审默认**：`/wx-on` 和 `/wx-switch` 默认免审，需手动 `/wx-approve 开启`
- **路径大小写**：项目路径比较忽略大小写，统一用正斜杠
